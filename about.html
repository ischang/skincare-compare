About this website

This wobsite was written in: vanilla Javascript, some Ajax/JQuery, HTML/CSS, and Twitter Bootstrap magic. The code lives here. It's hosted on Github pages and all the business logic is done completely client side.

Basically, that means nothing you input will get stored (except by your own local machine/browser) -- there's no server on my end or anything that's going to see what you typed, uploaded, or downloaded.

This tool also works with not just skincare products, but any product that has an ingredient list as long as you provide a .csv with the specified columns. So while I may call it a Skincare Compare tool since that's what I primarily use it for, it's basically just a .csv comparison tool (with the addition of Cosdna scraping).

Why I wrote this

This website began as a simple script to make my life easier. I have extremely sensitive skin that breaks out in extremely itchy, eczema patches when triggered by allergens or heat, so finding products that worked for my skin has always been extremely difficult.

Testing out products was always a mess of trial and error, loads of research, dozens of decant bottles, and ritual patch testing. I started a spreadsheet in late 2018/early 2019 and started comparing what products worked and what triggered my contact/atopic dermatitis and eczema. But as my list grew, comparing ingredients became a tedious chore.

I wanted a tool that could let me compare the laundry list of ingredients that each product seemed to have, but besides this Cosdna tool I found on r/AsianBeauty, which didn't work for me, there wasn't anything else.

So I wrote this. I was inspired by the tool above to add Cosdna scraping, but beyond that, the code is my own. It's not the prettiest and the logic and design is rather simple, but it made my life easier comparing ingredients. And, hopefully, it'll make your life easier as well. Happy holy grail and skincare hunting!

Bugs

If you're having problems with the tool, read the How To guide in detail and make sure your .csv file adheres to the proper format. If you're sure what you're experiencing is a bug, feel free to contact me below.

If you do decide to contact me, please give me your browser, browser version, the steps to reproduce the bug and, if you're able to, provide me with any console logs by going to the Developer Tools in your browser and clicking the 'Console' tab.

Alternatively, you can report the bug as an Issue on Github. If you know how to fix the bug, you can open a Pull Request.
